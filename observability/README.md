# Observability catalog

10 panels + 8 alerts per runtime module. Generated by `generate.py`,
validated by `validate.py`.

## Layout

```
observability/
├── dashboards/<module>.json   # Grafana 10+ schema (schemaVersion 39)
├── alerts/<module>.yml        # Prometheus / Mimir / Loki rule groups
├── generate.py                # source of truth — edit specs, regenerate
├── validate.py                # CI-style invariants
└── README.md
```

## Standard 10 panels

1. Request rate by route (RED)
2. Error rate by 4xx/5xx code (RED)
3. Latency p50 / p95 / p99 (RED)
4. Inflight requests (saturation)
5. CPU utilization (USE)
6. Resident memory (USE)
7. Active goroutines / tasks (saturation)
8. Tenant cardinality (multi-tenant guard)
9. Top-10 tenants by request volume (table)
10. Health probe (`up` per instance)

## Standard 8 alerts

1. `*SloBurnRateFast`        — 1h × 14.4 of a 99.9% target
2. `*SloBurnRateSlow`        — 6h × 6
3. `*ErrorBudgetLow`         — trailing 7d availability < 99.9%
4. `*LatencyP99High`         — p99 > 1 s for 5 m
5. `*SaturationHigh`         — inflight > 400 for 5 m
6. `*MemoryPressure`         — RSS > 4 GiB for 10 m
7. `*CardinalityExplosion`   — distinct tenants > 5000 for 30 m
8. `*HealthProbeFailing`     — `up == 0` for 1 m (critical)

Every rule carries:
- `severity: critical|warning|info`
- `tenant_scoped: "true"` where the rule's expression is per-tenant
- `runbook_url: https://docs.cave.dev/runbooks/<module>/<id>.md`

## Modules NOT covered — and why

These workspace crates are intentionally **out of scope** for the
observability standard because they don't expose a metrics endpoint of
their own (libraries, binaries, configs):

| Crate                | Reason                                            |
| -------------------- | ------------------------------------------------- |
| cave-core            | shared library                                    |
| cave-kernel          | shared library                                    |
| cave-db              | persistence library used by other crates          |
| cave-ebpf-common     | eBPF helpers library                              |
| cave-tracing         | OTel SDK — instruments other modules              |
| cave-cli / cavectl   | CLI binary                                        |
| cave-runtime         | runtime entrypoint binary (per-module dashboards) |
| cave-docs-site       | static documentation site                         |
| cave-docs            | docs library                                      |
| cave-changelog       | release-note library                              |
| cave-status          | status library                                    |
| cave-flags           | feature-flag library                              |
| cave-lint            | linting library                                   |
| cave-techdocs        | docs library                                      |
| cave-permission      | permission library                                |
| cave-gitops-config   | configuration shape                               |
| cave-portal-web      | static frontend (instrumented via cave-portal-api) |
| cave-cost-alloc      | cost-attribution library                          |

| cave-ledger          | accounting library                                |

## Regenerating

```sh
python3 observability/generate.py
python3 observability/validate.py
```

Both scripts are deterministic. Edit `MODULES` in `generate.py` to add
or rename a module — the validator will fail fast if a module is added
without a matching panel/alert pair.
