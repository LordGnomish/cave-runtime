## What `compact()` does (honest subset)

`compact(chunks, retention_cutoff_ns, out_codec) -> (Vec<Chunk>, CompactionStats)`:

1. Group decoded entries by `(tenant, stream_fp)` (BTreeMap ‚Üí deterministic order).
2. Retention: drop entries with `timestamp_ns < retention_cutoff_ns`; count expired.
3. Sort by `(ts, line)`, collapse exact `(ts, line)` duplicates; count deduped.
   Distinct lines sharing a timestamp are preserved.
4. A group emptied by retention emits no chunk.
5. Re-encode the survivors into one chunk per group with updated
   `min_ts`/`max_ts`/`num_entries`. Idempotent: re-compacting compacted output
   changes nothing.

Mirrors Loki's `pkg/compactor` table-compaction at entry granularity. The
multi-node table-shipper sync layered on top stays a manifest scope-cut
(single-process cave-logs).

## Coverage of the new test surface (`tests/compactor.rs`, 9 tests + 2 in-module unit tests)

- `merge_overlapping_chunks_same_stream` ‚Äî interleaved chunks merge, ts-ordered.
- `dedupe_identical_entries` ‚Äî duplicate `(ts,line)` across chunks collapses; count == 1.
- `dedupe_preserves_distinct_lines_same_ts` ‚Äî same ts / different line both kept.
- `retention_drops_expired_chunks` ‚Äî `ts < cutoff` dropped, `ts == cutoff` kept.
- `retention_emptied_stream_produces_no_chunk` ‚Äî fully-expired group emits nothing.
- `cross_stream_isolation` / `tenant_isolation` ‚Äî different fp / tenant never merge.
- `compaction_is_idempotent` ‚Äî second pass: 0 deduped, 0 expired, identical entries.
- `empty_input_yields_empty_output` ‚Äî `compact(&[])` is clean.
- in-module: `unit_merge_and_order`, `unit_dedup_count`.
