# RED phase

Marker file for the TDD RED phase of cave-vulns deep port.

The implementation of the deep port (finding/state machine, dedup with four
algorithms, CVSS v3+v4, 7 scanner parsers, risk_accept, sla, reports,
notifications, hierarchy, routes) is being implemented in this branch.

Each module contains a `#[cfg(test)] mod tests` block; those tests were
authored alongside the API surface they exercise. The contract:

  - finding::tests          — 19 tests covering state machine + serde
  - dedup::tests            — 11 tests covering 4 algorithms
  - dedup::scanner_fields   — 4 tests covering per-scanner field tuples
  - cvss::v3::tests         — 8 tests covering canonical FIRST vectors
  - cvss::v4::tests         — 7 tests covering MacroVector buckets
  - parsers::*::tests       — 7 parsers × 5-8 tests each
  - hierarchy::tests        — 5 tests for ProductType/Product/Engagement/Test
  - risk_accept::tests      — 8 tests for treatment + expiry + reactivation
  - sla::tests              — 8 tests for windows + rollup
  - reports::tests          — 6 tests for executive summary
  - notifications::tests    — 10 tests for sinks + fanout
  - routes::tests           — 11 tests for axum oneshot smoke

Total target: ≥150 new tests, all passing post-GREEN.

This marker file is deleted in the GREEN commit.
